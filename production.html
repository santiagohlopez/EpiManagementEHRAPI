Epi-Management API


INTRODUCTION


In this documentation we will list all the steps required to access protected health information based on the documentation of an open API.

Production

  • Authenticate
  • Search Patients
  • Search Encounters
  • Fetch CCDA

  • This api is used to validate the user and in return will give an access token along with other parameters.
    This access token will be used for all future API calls.Input here username and password in the body part.


    POST Authenticate

    https://neregaws.epilepsygroup.com/nereg.prod/rest/GetConnectionToken

    GET Token

  • HEADERS

    Content-Type
    application/json

    Content-Type
    application/x-www-form-urlencoded

    grant_type
    password

  • BODY Raw

    {
    "username":"*****",
    "password":"*****"
    }

  • Example Request

    curl --location --request POST 'https://neregaws.epilepsygroup.com/nereg.prod/rest/GetConnectionToken' \
    --header 'Content-Type: application/json' \
    --header 'Content-Type: application/x-www-form-urlencoded' \
    --header 'grant_type: password' \
    --data-raw '{
    "username":"*****",
    "password":"*****"
    }
    '
